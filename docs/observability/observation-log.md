# Observation Log

> **Owner:** Founder (you write entries) → Chief-of-staff (reviews, dispatches) → Agents (close the loop)
> **Rule:** Append-only. Add new entries at the top. Agents respond inline below each entry.

---

## How to add an entry (30 seconds)

```
## YYYY-MM-DD #needs-review
- [What you tested or observed]
- [What felt wrong or worth flagging]
- [Optional: which part of the system — booking flow, escalation, calendar, etc.]
```

Tag with `#needs-review`. Chief-of-staff picks it up, converts to a task, flips the tag to `#dispatched`. Agent resolves and flips to `#resolved` with a link to what changed.

---

## Status tags

| Tag | Meaning |
|-----|---------|
| `#needs-review` | Added by you — no agent has looked at it yet |
| `#dispatched` | Chief-of-staff has routed it to an agent |
| `#resolved` | Agent completed the work — artifact linked inline |
| `#watch` | No action needed now — monitoring only |
